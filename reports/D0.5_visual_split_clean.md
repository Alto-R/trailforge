## 1. T1.4' 彭晓视觉层 → `s_visual`

CLIP 已搁置（原图不可得，见 D0.4/计划）。改用彭晓 538k 场景分类点聚合到 50m 登山段。
每张照片归属其 **100m 内最近的段**（`sjoin_nearest`，避免重复计数）。

**输出**：`data_processed/segment_visual_climbing.parquet`，**29,941 段 × 54 维**：

| 组 | 维度 | 内容 |
|---|---|---|
| 密度 | 1 | `vis_photo_count` |
| main_cla | 4 | natural / manmade / natural&manmade / **indoor(噪声)** 占比 |
| med_cla | 13 | MT/FR/HB/TP/WT/PS/VB/LB/SC/CM/AL/WK/DW 占比 |
| fine_cla | 31 | Top-30 类 + other（mountain_path/canyon/forest_path/boardwalk/rope_bridge/temple/pagoda/ruin/creek… 户外性强，利于解释标签） |
| 季节 | 4 | 春/夏/秋/冬（由 month 推） |
| 标志 | 1 | `has_visual` |

**覆盖率（关键）**：538,848 点中 **122,455 (22.7%)** 落在登山网络 100m 内；**仅 9,179 段 (30.7%) 有照片**。
→ 约 69% 的段 `s_visual` 全 0（由 `has_visual=0` 标记）。CF 片段塔须能容忍视觉层大面积缺失（`has_visual` 掩码 + 优雅退化）。密度：覆盖段中位 4 张、均值 13.3、max 1256。

> 维度比原 CLIP 方案（~500 维 embedding）小一个数量级（~54 维），CF 片段塔输入相应缩小——这是放弃 CLIP 的直接后果，已知并接受。

## 2. T1.2 交互矩阵 train/test 划分（档案B）

按 §5.3.2 **每用户时间维度 80/20**（最早 80% 训练 / 最近 20% 测试）；纯档案B 操作，不触档案A 特征（保持双档案隔离）。`first_ts` 0% 缺失。

**输出**：`interactions_train.parquet` / `interactions_test.parquet`

| 指标 | 值 |
|---|---|
| train / test 行 | 1,276,886 (80.1%) / 317,364 (19.9%) |
| 有测试正样本的用户 | 3,578 / 3,630 (98.6%) |
| 测试正样本中"新段"（训练未见） | 274,638 / 317,364 (**86.5%**) |
| 时间违规（test 早于该用户 train 最大 ts） | 0 |

86.5% 测试正样本是训练未见的新段 → 适合做 ranking 评估的真实预测目标。
