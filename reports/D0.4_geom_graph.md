## 3. TrailGraph（T1.0，已实现 + 测试通过）

[src/trail_graph.py](../src/trail_graph.py)：按端点 snap（1m）建段级邻接 `dict[lypID, list[lypID]]`，缓存 `data_processed/adjacency_climbing.pkl`。接口含 `neighbors / attributes / subgraph(radius_km, 图距离) / shortest_path(Dijkstra) / find_nearest_segment(空间索引)`，对齐主文档 §5.4.1。

**连通性统计**：

| 指标 | 值 |
|---|---|
| 段数 | 29,941（全部有邻居，deg_zero=0） |
| 平均度 | 2.143 |
| 最大度 | 5（snap≤10m）/ 7（snap=20m） |
| **连通分量数** | **610** |
| 最大分量 | 4,478 段（≈224km） |

**关键发现（非 bug）**：连通性在 snap 0.5m→10m **完全不变**——说明 610 个分量是网络的**真实属性**，不是 snap 容差伪影。北京登山步道天然分裂为多个互不相连的山块（西山/香山/八达岭…）。

**对下游的含义**：
- 路径生成天然被限制在单个连通分量内——这符合"一条路线待在一个区域"的直觉，不是缺陷。
- 主力区是最大分量（4,478 段）；`find_nearest_segment` + `subgraph(radius)` 会自然落在所属分量内。
- 平均度 2.14（vs 原 workingdata5 的 3.15）：本网络是 29,941 段的春季登山子集，近线性步道为主，属正常。

**测试**：[tests/test_trail_graph.py](../tests/test_trail_graph.py) 8/8 通过（邻接对称、度数有界、属性含 geo 字段、直连最短路、跨分量返回空、subgraph 半径单调、nearest 命中）。环境无 pytest，测试写成纯 stdlib 可直接 `python tests/test_trail_graph.py`。
