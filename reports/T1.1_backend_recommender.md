# T1.1 后端推荐服务（探索辅助形态）

工程线第一块交付：把研究成果做成可调用的 `/route` 服务。源码：[backend/](../backend/)（app/engine/schemas/feedback）、[src/cf_export.py](../src/cf_export.py)、[src/persona.py](../src/persona.py)、[src/diversifier.py](../src/diversifier.py)、[src/explainer.py](../src/explainer.py)、[src/route_generator.py](../src/route_generator.py)。设计与演进：[docs/plans/2026-06-02-backend-engine-design.md](../docs/plans/2026-06-02-backend-engine-design.md)。

## 1. 决定性发现：E4 内容塔不个性化（冷启动）

初版打算"persona→内容塔评分→单条个性化路线"，跑通后诊断推翻了该前提：

| 诊断 | 结果 |
|---|---|
| 5 个 persona 的 g_u 嵌入两两 cosine | 0.999–1.0 |
| **真实 603 用户**（真实软 cluster+llm）g_u 嵌入两两 cosine | 均值 **0.9995** |
| 6 个随机真实用户评分向量两两 Pearson | **1.0** |
| persona c0 评分 vs 段 beh_cluster_0 相关 | -0.325（未学到匹配） |
| 12 起点 × 5 persona 的 2.5km 最高分目的地 | **全部相同** |

**结论**：`g_u(任何用户) ≈ 同一个常量查询向量` → 冷启动对所有用户给出**完全相同的段排序**。E4 的个性化全在**协同路径**（user_id 嵌入）；内容塔在冷启动只是"按段流行度/特征"的**全局先验**。研究里"冷启动内容 2.5× 协同（0.457 vs 0.182）"应重述为 **"流行度/特征先验 > 随机未训练嵌入"，而非个性化推荐**。

## 2. 决策：按主计划 §4.5 转为探索辅助形态

主计划早已定产品为**探索辅助而非精准推荐**，并写明用"多候选+排序+可解释"对 CF 不精准容错。上述发现**印证而非推翻**该选择。个性化信号来源（用户确认）：

- **显式偏好**主导可见个性化：persona→数据派生默认偏好权重 + 滑块可调。验证 **challenge vs scenic+nature 在 19/20 起点走出不同路线**（PREF_MAP：challenge→坡度 / nature→自然 / culture→人文 / popularity→流量 / scenic→打卡）。
- **CF 段分**作相关性/质量先验（跨用户恒定，但学习式融合 83 维）。
- **MMR**（§5.4.5，β=0.7、Jaccard）给多样性 + **近重复早停**（凑不满 n 条就少返，诚实）。

## 3. 架构与接口

请求流：`偏好(滑块>persona默认>均衡) → 段分=W_pref·偏好属性 + W_cf·CF先验 → generate_pool(方向播种候选池) → MMR → 每条候选+geojson+属性+可解释标签`。全部产物从 `data_processed/` 加载，**不依赖 G: 盘**（新增 `segment_geom_climbing.parquet` 几何缓存 + `segment_logcnt_climbing.parquet`）。

> **融合权重当前为固定硬编码**（[engine.py](../backend/engine.py)）：`W_pref=1.0`、`W_cf=0.3`，不读 config、不随请求变、用户不可调。`W_pref > W_cf` 落实了主计划 §4.5「显式偏好主导可见个性化、CF 仅作先验」的定性方向，但 **plan 未规定具体数值，这两个值是落地时的工程取值**，属待调整项（见 §5）。注意与**模型层 adaptive α(u,s)**（`sigmoid(w0+w2·log_cnt_s)`，融合 content/collab 塔，是学出来的）区分——后者自适应，前者固定。

| 接口 | 说明 |
|---|---|
| `GET /health` | 模型/数据就绪（n_segments=29941, n_personas=5） |
| `GET /personas` | 5 个 persona + default_prefs |
| `POST /route` | 入 `{start:[lng,lat], persona?, preferences?, budget_km, n_routes}`；出 2–5 条 MMR 候选（length/segments/geojson/attributes/labels/score）+ start_snapped + note |
| `GET /trails` | 全网 GeoJSON（610 碎片，引导点选） |
| `POST /feedback` | 记录选了哪条 + 评分 → jsonl（Module E 最小实现） |

技术栈 FastAPI；E4 作 CF 先验由 [cf_export.py](../src/cf_export.py) 全量训练一次存盘（活跃域 R@10=0.506，对齐研究 0.508）。
